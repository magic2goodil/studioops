# StudioOps Plugin Initial Submission

StudioOps is a skills-only Codex plugin for installing and operating a local-first AI software delivery workflow.

This initial developer-preview submission includes:

- one-step local setup for the current repository;
- structured project, epic, and task intake;
- user stories, acceptance criteria, dependencies, attachments, and project standards;
- isolated builder and specialist-review workflows;
- non-production QA and a human-controlled release boundary;
- explicit privacy, security, and truthful-status safeguards.

The plugin does not include an MCP server, hosted StudioOps account, production deployment authority, or credentials. It installs or communicates with the user's local StudioOps Community control plane. Review tests use local synthetic repositories and require no test account or private fixture data.

StudioOps remains a developer preview. The listing and documentation disclose its local storage model, optional Codex and GitHub interactions, current platform limitations, and best-effort community support.
